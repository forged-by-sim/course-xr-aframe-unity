1. Model Invisible in Scene (A-Frame)

The very first issue was that the Rhomaleosaurus model simply never appeared, even though:

The red cube appeared,

The grey plane appeared,

The scene rendered normally,

And there were no syntax errors.

Attempts made:

Adjusted model scale from extremely small values to extremely large ones.

Moved the model closer and farther on all axes.

Rotated the model in multiple directions.

Toggled visible on and off.

Rewrote the gltf-model path.

Tried .gltf and .glb versions of the model.

Used lighting adjustments to see if the model was lost in darkness.

Checked A-Frame Inspector to confirm the entity existed.

Outcome:
No model appeared. This was the first sign that the model was not being loaded at all.

2. File Dragging Behaving Incorrectly

When attempting to drag .glb files into Chrome for testing, the browser simply downloaded the file again instead of previewing it.

Repeated attempts:

Dragging into a new Chrome tab,

Dragging into an existing tab,

Trying Edge,

Trying Firefox.

Outcome:
All browsers downloaded the file instead of displaying it. This suggested Windows did not have proper MIME handling for .glb.

3. Opening the GLB in Chrome Failed

Double-clicking the .glb file:

12. Orbit Controls Not Working Initially

Early attempts using orbit controls produced:
